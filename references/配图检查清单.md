# 配图检查清单

这份清单用于在正式导出图片前，检查方案是否已经达到可发布状态。

目标不是“挑毛病”，而是避免把明显方向错误、截图模糊、版式截断的图片继续推到最终稿。

## 一、图任务一致性检查

先检查标题、主体、底部说明是否在讲同一件事。

必须满足：

- 主标题表达的是这张图真正的核心认知
- 主体截图 / 图形展示的是与标题同一件事
- 底部说明解释的是主体内容，而不是切换成另一种叙述方向

不通过的常见情况：

- 标题在讲“字段作用”，底部却在讲“使用流程”
- 标题在讲“安装步骤”，主体却主要展示字段说明
- 主体是截图证据，底部却塞入了正文总结

只要标题、主体、底部三者中有任意一项不在同一个图任务下，这张图就必须回退重做。

## 二、截图质量闸门

如果当前方案使用了截图，必须逐项检查：

- 截图是否清晰
- 关键区域是否足够大
- 手机上看时关键文字是否可读
- 是否还残留无意义遮挡，例如时间标签、头像边角、字幕遮挡
- 是否已经裁掉无关区域
- 是否需要局部放大
- 是否应该添加箭头、编号或高亮框

以下情况不允许直接发布：

- 关键区域太小
- 细字代码无法辨认
- 整帧截图里只有一小块内容真正有价值
- 截图中存在明显遮挡
- 截图模糊到只能当“氛围背景”

如果以上任一情况存在，应优先：

1. 裁切
2. 放大
3. 标注
4. 重绘为信息图

而不是继续直接使用整帧截图。

## 三、图片文字检查

图片中的文字必须是“图用语言”，而不是正文压缩版。

必须满足：

- 主标题不宜过长，避免难看换行
- 副标题只补充一层解释
- 标注数量控制在 3 到 5 个
- 每个标注只讲一个点
- 不要出现大段段落

以下情况说明图片文案还没准备好：

- 图中文字比正文还难读
- 读者不看正文也觉得啰嗦
- 一张图同时想讲太多信息

## 四、画布与导出检查

正式导出前，必须检查：

- 底部内容是否完整显示
- 是否存在大面积无意义留白
- 画布高度是否与内容高度匹配
- 是否因为标题太大导致正文被压缩
- 手机上看时字是否过小

以下情况应视为导出失败：

- 底部模块被截断
- 下半部分大面积空白
- 标题字号过大导致观感失衡
- 内容虽然完整，但在手机上阅读明显吃力

## 五、最终发布判断

只有在以下条件都满足后，这张图才应进入最终稿：

- 图任务一致
- 截图质量过关，或已改为非截图方案
- 文案简洁，且没有复述正文
- 画布没有截断，也没有明显留白问题
- 读者能在 3 秒内知道这张图要表达什么

如果仍然存在 2 项以上风险，这张图不应进入最终发布稿。
