# 视频分析稿模板

高质量视频理解模式下，第一阶段默认产物应是“视频分析稿”，而不是纯逐字稿。

目标：

- 让后续知乎改写不再只依赖音频文字
- 把“讲了什么”和“做了什么”同时沉淀下来
- 为截图、片段、表格、代码块等后续可视化提供结构化输入

## 推荐文件头

```markdown
---
type: content
format: analysis
status: draft
area: "[[写作]]"
created: YYYY-MM-DD
tags: [content, 视频分析, 多模态, 知乎]
source: "<原链接>"
---
```

## 使用原则

### 1. 这不是逐字稿

- 不要把口播按顺序抄下来就当分析稿
- 必须做结构化归纳
- 必须加入操作层面的观察

### 2. 这也不是最终发布稿

- 分析稿允许保留“待确认”
- 允许保留更偏内部加工的结构
- 但不能过度口语化，也不能只是一堆散点笔记

### 3. 优先服务后续改写

分析稿里最重要的不是“完整”，而是“可复用”。

后续知乎改写最依赖的是：

- 主轴判断
- 重点排序
- 操作时间线
- 事实卡片
- 不确定信息

此外，工作稿和最终稿都应使用视频分析稿中的证据等级来决定：

- 哪些信息可以直接写成事实
- 哪些信息只能保守表达
- 哪些信息必须先问用户

### 4. 视觉素材建议是可选项

不是每条视频都必须有截图或片段建议。

只有这些情况才建议补：

- 单靠文字难以说明的操作
- 入口位置、页面切换、代码对比本身就是重点
- 动态变化本身就是信息

如果没有明显必要，可以直接留空。
