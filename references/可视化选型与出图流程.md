# 可视化选型与出图流程

AI 根据正文内容信号自主判断：是否出图、出什么类型的图、如何生成。

**不使用预设模板**。图类型由内容决定，不由模板驱动。

---

## 第一步：判断是否出图

只有正文中出现以下任一内容信号，才进入出图流程：

| 内容信号 | 说明 |
|---|---|
| 两种方案 / 方法 / 路径的明确对比 | 如"旧方式 vs 新方式""单模型 vs 多模型" |
| 3 个以上角色、模块或工具的分工 | 如多 Agent 协作、工具链分工 |
| 3 步以上的线性操作或闭环链路 | 如安装流程、内容生产链路 |
| 3 到 4 个值得读者记住的核心结论 | 如方法总结、经验判断 |
| 靠纯文字难以快速说清的结构关系 | 如模块依赖、信息流转 |
| 明确的量化数据或趋势对比 | 如效率变化、使用前后对比 |
| 操作演示型内容中存在界面定位、按钮位置、状态变化 | 如"点右下角工具栏""拖拽到书签栏""暂停后状态切换"，纯文字读者难以跟随操作 |

**不满足以上任一条件，不出图。**

---

## 第二步：根据内容信号选图类型

### 对比图

**触发信号**：正文中明确出现两种方案、两种方法、两种路径的对比，且差异是文章重点。

例如：
- 单模型 vs 多模型协作
- 手工流程 vs AI 辅助流程
- 优化前 vs 优化后的工作方式

**不触发**：只是顺带提到两个工具，没有明确比较。

### 角色分工图

**触发信号**：正文中出现 3 个左右角色、模型或模块，且每个都有明确职责。

例如：
- 三个 Agent 各司其职
- 用户 / 工具 / 数据三层关系
- 内容生产中的创作者 / 工具 / 平台

**不触发**：只是列举了几个工具名字，没有说明分工关系。

### 流程图

**触发信号**：正文中存在有明确前后顺序的操作路径，或存在回流、闭环结构。

例如：
- 输入 → 处理 → 输出的链路
- 主 Agent → 子 Agent → 审核 → 输出的闭环
- 安装 → 配置 → 使用的步骤路径

**不触发**：只是泛泛描述了做了几件事，没有明确顺序依赖。

### 步骤图

**触发信号**：正文中有 3 到 5 个线性步骤，且步骤顺序对读者理解有帮助。

例如：
- 三步安装教程
- 四个配置阶段
- 五步内容改写流程

**与流程图的区别**：步骤图是线性单向的，流程图可以有分支、回流或闭环。

### 总结图

**触发信号**：正文中有 3 到 4 个值得读者单独记住的判断或结论，且这些结论是文章的核心交付。

例如：
- 三个使用建议
- 四个注意事项
- 三个核心判断

**不触发**：只是正文末尾的普通总结段落，没有独立的结论价值。

### 数据图

**触发信号**：正文中出现明确的量化数据、比例、趋势或前后对比数字。

例如：
- 效率提升百分比
- 使用前后耗时对比
- 多个方案的量化差异

**不触发**：只是定性描述，没有具体数字。

---

## 第三步：出图前写图任务定义

每张图在生成前，形成一份极短 brief：

- **设计方向**（内部锚点，不出现在图上）：这张图要让读者理解 \_\_\_\_\_\_
- **图类型**：
- **主标题**：（图内展示，20 字以内，用"图用语言"）
- **主体核心**：图的中间区域放什么
- **关键标注**：2 到 5 个（每条不超过 20 字）
- **收口**：是否需要底部总结（主体已足够则不加）
- **技术路线**：HTML/CSS 还是 Mermaid + HTML 容器

**关于「设计方向」的约束**：

- 设计方向是 AI 内部的设计锚点——它决定布局选择、数据取舍、视觉重心的位置
- **禁止将设计方向直接复制为图内标签、标题或标语**
- 如果设计方向写不出来，说明还不该出图
- 图上实际展示的标题和标注必须用具体的、可感知的"图用语言"重新提炼

---

## 第四步：选择技术路线

### 走 HTML/CSS

适合所有"卡片感"图：对比图、角色分工图、步骤图、总结图、数据图。

约束：
- 不做整页网页感布局
- 不把正文段落大面积塞进卡片
- 一张图只讲一个核心认知
- 主体足够时不强加底部收口区

### 走 Mermaid + HTML 容器

适合有明确节点关系或闭环结构的流程图。

约束：
- Mermaid 不裸输出，必须包在 HTML 容器里
- HTML 容器提供标题区和背景，Mermaid 负责逻辑结构

---

## 第五步：出图后自检

生成代码后，回答以下问题：

- 这张图有没有超过一个信息中心？
- 底部是否有大面积空白？
- 标题是否一眼就能说清图在讲什么？
- 在手机宽度下，关键文字是否仍可读？
- 这张图是否真的帮读者理解了正文中的某个难点，还是只是重复了正文？

有任一问题答不上来，需要重新设计。

---

## 插入位置建议

- **对比图**：放在正文分析"差别、变化、收益"之前
- **角色分工图 / 流程图**：放在讲"怎么运作、为什么这样做"的段落附近
- **步骤图**：紧跟步骤说明段落之前或之后
- **总结图**：放在结尾前，帮助读者收藏
- **数据图**：放在引用数据的正文段落之前

---

## 数量控制

一篇知乎稿默认不超过 3 张核心配图。文章较短时只做 1 到 2 张。

优先级：先做最能帮读者理解核心内容的那张，再考虑补充图。
