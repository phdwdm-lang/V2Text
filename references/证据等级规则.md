# 证据等级规则

做视频转图文时，最大的风险不是“写得不顺”，而是把**推断**写成**事实**。

这份规则用于约束：

- 视频分析稿如何标记关键信息的可信度
- 工作稿如何处理不同证据等级的信息
- 最终发布稿哪些内容可以写实，哪些必须补问或删除

## 核心原则

不是所有信息都处于同一个确定性层级。

同一条视频里的信息，通常会来自：

- 用户补充
- 画面可见内容
- 口播内容
- 多模态综合推断
- 明显不稳定的识别结果

如果不区分来源级别，后续就很容易把：

- “看起来像”
- “大概率是”
- “视频里似乎提到了”

直接写成：

- “就是”
- “已经确认”

## 推荐等级

### A. 用户确认

定义：

- 用户在对话中明确确认过的信息

写法：

- 可以在最终稿中直接当作已确认事实使用

### B. 画面可确认

定义：

- 不依赖口播，仅凭画面或截图就能较稳定确认的信息

写法：

- 可以进入工作稿
- 进入最终稿前，若属于关键事实，最好再通过用户确认或补充链接加固

### C. 口播可确认

定义：

- 主要来自口播或转写，内容本身清晰、稳定、歧义较小

写法：

- 可以进入工作稿
- 如果涉及专有名词、链接、命令、仓库名，不能只凭口播直接写入最终稿

### D. 多模态推断

定义：

- 由画面、口播、上下文综合推断出来，但不能完全确认

写法：

- 可以出现在视频分析稿里
- 工作稿里必须降级写法
- 最终稿里应尽量转化为更保守表达，或在必要时先问用户

### E. 待确认

定义：

- 当前证据不足，或者识别明显不稳定的信息

写法：

- 允许出现在视频分析稿里
- 允许进入提问清单
- 不应直接进入最终发布稿

## 不同阶段如何使用

### 视频分析稿

- 必须允许保留 A/B/C/D/E 全部等级
- 但关键事实最好标注来源级别

### 知乎工作稿

- 可以使用 A/B/C
- D 级信息必须降级表达
- E 级信息原则上不直接写入正文，应转去问题清单或备注区

### 知乎最终稿

- 优先只保留 A/B
- C 级信息如果不是关键事实，可保留
- D 级信息只有在不会误导读者时，才能以保守表达保留
- E 级信息不得进入最终稿

## 关键事实的默认范围

以下内容只要出现，就应被视为关键事实：

- 工具 / 产品正式名称
- 仓库链接
- 安装入口
- 命令
- 快捷键
- 网站 / 平台名称
- 文件路径 / Skill 名称

这些内容如果证据等级不足，不应写实。

## 推荐标记方式

在视频分析稿里，建议用这种形式：

```markdown
- 名称：Selector
  - 证据等级：A 用户确认

- 安装入口：浏览器书签栏
  - 证据等级：B 画面可确认

- 输出字段完整列表
  - 证据等级：E 待确认
```

如果不想每条都显式写“证据等级”，至少要在“不确定信息”里明确列出低确定性的关键点。
