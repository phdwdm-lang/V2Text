# 术语纠错规则

这个 Skill 处理的是 AI 工具、Agent、vibecoding、产品设计类内容，转写时最容易错的是英文产品名、命令行词汇和缩写。

## 高优先级术语

优先把明显误识别修正为以下标准写法：

- Claude
- Codex
- Gemini
- Cursor
- Claude Code
- Claude CLI
- Markdown
- SVG
- API Key
- Agent

## 常见误识别模式

以下是当前观察到的高频误识别，修正时优先参考：

| 误识别 | 目标写法 |
|---|---|
| `cloud` | `Claude` |
| `CLOLOLOD` / `COUDOD` / `CLOLOUD` | `Claude` |
| `cloud x` / `code x` / `code 1` | `Codex` |
| `German` / `Gerald` / `gmt` | `Gemini` |
| `SPG` | `SVG` |
| `Malcom` | `Markdown` |
| `APIK` | `API Key` |
| `Crosse` | `Cursor` |

## 修正原则

### 1. 优先修正品牌名

如果一句话里同时出现“写方案、写代码、做设计”这类模型分工语境，通常对应的是：

- `Claude`：偏方案、理解、产品思考
- `Codex`：偏代码执行
- `Gemini`：偏视觉、设计、补充方案

### 2. 先保语义，再保逐字

如果逐字稿中的英文串已经明显不可读，不要勉强保留原词。优先恢复正确术语，让后续图文改写可用。

### 3. 无法确定时用中文降级

如果个别缩写无法百分百确认，优先改成含义明确的中文表达，例如：

- `Markdown 文件`
- `命令行工具`
- `设计模型`
- `官方 API`

不要把明显错误的英文继续带入知乎正文。

## 不要过度清洗

不要把所有英文都中文化。以下内容通常应保留原文：

- 产品名
- 命令名
- 技术术语
- Markdown / HTML / JSON / API 等标准词

## 二次改写前的检查清单

在把逐字稿改写为知乎图文前，快速检查：

- 是否还有 `cloud / German / code x` 这种残留
- 是否有大小写明显错误的品牌名
- 是否有标点导致的英文粘连
- 是否出现“能猜到但不专业”的术语拼写
